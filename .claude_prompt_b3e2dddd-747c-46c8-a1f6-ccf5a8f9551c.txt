**Title**: [PR Feedback] ELK Layout Engine
**Description**: Based on consolidated PR review, address the following in **ELK Layout Engine**:

- **`stripPortSuffix` Returns Empty String for Undefined Port IDs**: When `portId` is `undefined`, `stripPortSuffix()` returns `''` at `/workspace/src/core/layout/engines/ELKLayoutEngine.ts:345`. This empty string is then used as `source` or `target` at lines 411-412, creating phantom edges referencing a non-existent node with ID `''`. Should return `undefined` or the edge should be skipped.
- **`ELKEdgeRouting` Type Missing `'NONE'` Value**: The type at `/workspace/src/core/layout/engines/ELKLayoutEngine.ts:48` defines `'ORTHOGONAL' | 'POLYLINE' | 'SPLINES' | 'UNDEFINED'` but the default is set to `'NONE'` at line 221. Users of `ELKParameters.edgeRouting` cannot set `'NONE'` via the typed API despite it being the runtime default.
- **`edgeRouting: 'NONE'` Removes ELK Fallback Routing**: Setting `'NONE'` at line 221 means ELK never populates `edge.sections`. If Libavoid WASM fails to load, edges get no routing from either system and fall back to per-render A* pathfinding in `ElbowEdge` — the exact performance bottleneck this PR replaces. Previously ELK provided a middle-ground fallback. `/workspace/src/core/layout/engines/ELKLayoutEngine.ts:221`
